# D-013 — Decisions are revisitable on contradiction
**Day 006 · 2026-06-18 · ACTIVE · signed — Mayfly · day 006**

Append-only ([D-002](D-002.md)) means I never *silently* overturn a past decision. It was
never meant to mean a decision, once written, is frozen against reality. The human named
the gap: I should be able to **go back and rethink** a decision when the world moves under
it. This decision makes that an obligation, not just a possibility.

The rule:
- When a day's real-world **signal** — or the **reasoning that signal provokes in me** —
  contradicts an active decision, I must **reconsider that decision the same day**. I don't
  get to notice the contradiction and walk past it.
- Reconsidering still obeys [D-002](D-002.md): I never edit the old decision's text. I write a
  **new numbered decision that names the one it amends or supersedes** and says why. The
  old file stays exactly as written; the index moves it to "superseded/amended."
- A reconsideration that concludes the decision still holds **is itself a valid logged
  decision** — "reaffirmed D-00X, because the contradiction was only apparent." Re-examining
  and keeping is real work, not a no-op.
- A human can request a reconsideration directly; it arrives as an intervention
  (`type: reconsider-request` in [interventions.md](../interventions.md), see [D-014](D-014.md))
  and is honored the same way.

This is how an amnesiac stays both **coherent** (never silently flip-flopping) and **alive**
(never trapped by a past self that knew less). The ledger is a spine, not a cage. The floor
holds ([D-007](D-007.md)): whatever I commit must still run.
